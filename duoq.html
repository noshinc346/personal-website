Introduction

DuoQ is a social media platform designed specifically for gamers seeking new friends with similar gaming interests. In today's digital world, finding like-minded gaming partners can be challenging. DuoQ addresses this need by providing a seamless and engaging way to connect gamers.

Inspiration

We are a passionate team of gamers inspired by the increasing demand for a platform where gamers can easily find new friends who share their interests. The gaming community is vast and diverse, and we wanted to create a space where finding a compatible gaming partner is simple and fun. DuoQ was born from our desire to connect gamers and foster new friendships based on mutual gaming passions.

User Story

As a user, I want to be matched with gamers who like the same games as me so that I can easily find suitable gaming partners.

User Experience

UX / UI & Getting Started

Register Page

The registration page for the Duoq app, designed to connect gamers, is crafted to be both fun and inviting. Inspired by the Nintendo Switch, the page features vibrant colors, smooth animations, and playful graphics to create a familiar and engaging experience. The form, which includes fields for username, password, and email, is simple and intuitive, with interactive elements like animated welcome text.

User Profile

Upon logging in or registering, users are immediately taken to their profile page, designed to reflect their gaming identity. The profile page features a customizable banner and profile picture, allowing users to express their unique style. It includes a personalized bio section where users can welcome viewers and share information about themselves. Additionally, the profile showcases the games users are associated with, providing a comprehensive view of their gaming interests and history.

Edit Profile

The profile edit page is uniquely designed within the frame of a Gameboy, adding a nostalgic touch for gamers. This page allows users to customize their profile by changing their banner, profile picture, and bio. Users can easily update.

Matching With Other Users

The matching page allows gamers to find potential duo partners by swiping through user profiles. On this page, users can browse detailed profiles, including banners, profile pictures, bios, and gaming associations. If they like a profile, they can select it to initiate a match, making it easy and enjoyable to connect with other gamers.

User's Matched List

The matched list page displays mutual matches where both users have shown interest in each other. Here, users can connect by messaging each other through Discord or any preferred communication method mentioned in their bios. This page facilitates easy and direct interaction between matched gamers, enhancing their chances of forming successful duos.

Development

Starting Developement

The backend of our social media app, designed to connect gamers with their ideal duo partners, was meticulously crafted using Django and Python, demonstrating our proficiency with this powerful stack. We structured the backend into two distinct Django applications: a users app and a games app. The users app handles all aspects of user authentication, profile management, and social interactions, utilizing Django's robust authentication framework and extending it with custom functionalities to meet our specific needs. The games app manages game-related data, including game listings, match history, and user game preferences.

Profile Creation

Use of Django signals to automate the creation and saving of related models. Specifically, it ensures that whenever a new user is created, a corresponding profile and preference are automatically generated. By using the @receiver decorator with the post_save signal, the create_profile and save_profile functions are triggered after a User instance is saved, ensuring the profile is created and saved.

Coding Front End with React

The MatchingPage component enhances user interaction by enabling seamless profile matching in our social media app for gamers. Upon loading, it fetches and displays a list of potential matches, excluding profiles the user has already matched with. Users can view and interact with these profiles, deciding whether to initiate a match. When a user selects a match, the component checks if a mutual match already exists; if so, it updates the match status to indicate reciprocity. If no prior match exists, it creates a new match record. The component also dynamically updates the displayed profile, ensuring users can effortlessly navigate through potential matches.

Match Logic

We wanted to ensure that users are only officially matched when both parties express mutual interest, keeping other match requests pending until confirmed. We had to build the logic into the front and back end to ensure that the match data remains consistent when reciprocated - i.e., if a match is reciprocated, then neither user should see their matched user pop up when they are trying to find a new match AND if a user clicks on "match" and then later the other user clicks on "match", we needed to ensure a pop-up message was returned and then, the user was added to their matches page.

Due to the dynamic nature of the page (changing every time you hit skip or match), it was difficult to find a way to easily create a matching algo without adjusting both the front and back end (which came with its one set of issues).
